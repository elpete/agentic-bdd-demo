# Agentic ColdBox Guidelines

Use these rules when generating or editing code in this repo.

- This is a ColdBox 8 + BoxLang app generated by CommandBox.
- App code lives under `app/`; tests live under `tests/specs`.
- Use `.bx` for BoxLang classes and `.bxm` for templates.
- Avoid reserved scope names such as `session`, `application`, `request`, `url`, `form`, `variables`, and `arguments` for local variables or function arguments.
- Prefer public behavior over private implementation details.
- TestBox specs should use `describe`, `it`, and `expect`.
- Use clear Given / When / Then comments when they make behavior easier to scan.
- Do not over-mock domain objects; build real `Session`, `Speaker`, `Review`, and `ScoreCard` instances unless a collaborator boundary is explicit.
- Use targeted TestBox runs while iterating:
  - `box run-script test:target`
  - `box run-script test:unit`
  - `box testbox run bundles=tests.specs.unit.SessionDecisionServiceSpec outputFormats=mintext`
- TODO: `box testbox run options:dryRun=true` reaches TestBox 7 dry-run discovery, but CommandBox 6.3.2's CLI result parser currently expects execution totals. Use `box run-script test:dry` or the browser URL in the runbook until the CLI syntax/parser is updated.
- Treat tests as source-of-truth evidence. If a spec fails, diagnose whether the spec or implementation is wrong before changing either.
